Block 1 Functions and Mathematical Modeling

  1. Intro to the course

    1. NTI
      1. We can make the point about Linear Function is the model for change by drawing pictures or discussing how certain quantities change between two points in time
      2. The discussion about Calc Z should not be about “How is Z different,” but “Z is the new Calculus to which USAFA is transitioning; you may see differences between this course and the traditional course that some of your peers are taking.”
      3. Oveall goal for day 1 is to build cadet’s confidence that they can be successful. This requires us to show them the systems, resources, philosophy, and expectations (e.g., college \(\neq\) high school) behind the course.
    2. Lecture Topics
      1. Calculus Z overview and trajectory
      2. Course resources and systems
      3. Calculus is the study of change
      4. Linear Function is the model for “change” because it has “slope”
    3. Outcomes
      1. Understand than an equation relates quantities, but a function maps inputs to outputs
      2. Understand the difference between \(\equiv\) and \(=\)
      3. Be able to identify a function’s input
    4. Readings (assigned M1, skim before, read after)
      1. Chapter 1
    5. Additional Resources
      1. None
    6. Assignments
      1. HW01
      2. Feedback01
      3. DD01

  2. Basics of modeling functions

    1. NTI
      1. Emphasize that functions map inputs to outputs, but the naked functions are (by themselves) not flexible enough for most circumstances
      2. Show students graph of 5 naked functions, have them pick which is which
      3. Points out that a truly naked straight-line is just \(x\), and use that as the introduction to parameterization
      4. Show scatter plot of \(f(x)\equiv -x+2\) and explain why \(f(x)\equiv x\) is not a good model to motivate basic modeling function
    2. Lecture Topics
      1. Naked modeling functions
      2. Basic Modeling functions and their behaviors based on their parameters
    3. Outcomes
      1. Know graphs, domains, ranges, horizonal/veritcal intercepts & asymptotes (if any) for all the naked modeling functions
      2. Recognize that functions are a way of modeling relationship in the real-world
      3. Given parameters, describe how those parameters affect the shape of the function
    4. Readings: Section 3 and Chapter 2
    5. Additional Resources
      1. PDF - Exponential Functions Deep Dive
      2. Math Insight - Logarithm basics
      3. Khan Academy - Properties of Logarithms
      4. RESOURCE FOR TRIG VALUES (TRIG TABLE)
    6. Assignments
      1. DD02

  3. Functions as I/O and Notational Structure of functions (R<>Math)

    1. NTI
      1. Rewrite \(y=mx+b\) as \(mx-y+b=0\) to show function notation eliminates ambiguity of IO
      2. Break student habit of using “x-axis” or “y-axis”; describe these as “horizontal” or “vertical” axes
      3. Break student habit of using “x” and “y”; instead they should say “input” and “output”
      4. Explain that parameters scale or shift inputs/outputs. Ex: the \(C\) in \(Ce^{kt}\) scales the output and the \(k\) scales the input
      5. Example function without a formula: \(f:Cadet\to RoomNumber\)
    2. Topics
      1. Functions as Input and Ouptut (IO)
      2. Introduce Multi-input functions
      3. Function Representations
      4. Notation and Conventions
      5. Distinguish between mathematical symbols: \(=\), \(\equiv\), $, <- See Secton 5.8
    3. Outcomes
      1. Understand that a function maps inputs to outputs
      2. Given an input, be able to find the output of a function in any one of the five standard forms:
        • Graphs of functions (including contour plots)
        • Tables of inputs and outputs
        • Word descriptions
        • Computer function name
        • Mathematical notation
      3. Be able to translate between traditional math notation and R expressions
    4. Reading: Chapter 5 & 6
    5. To do:
      • Provide glossary of notation across 6-blocks (+alternate forms)
    6. Assignments
      1. DD03

  4. Parameterized functions as models

    1. NTI
      1. Introduce idea that input has meaning, and output has meaning relative to the input
      2. Give students a function. Then change the units of input to a different unit (e.g. feet go to Meters)
      3. Logarithms are not necessarily used as the end model; they are the lever which seperate power from exponential functions
        • TODO: Think about physical phenomenon that would appropriately be modeled by a Logarithmic function or power with \(p<1\)
      4. Focus the lesson entirely on finding parameters for Sine, Power, and Exponential
    2. Lecture Topics:
      1. Modeling phenomenon with the basic modeling functions
      2. How to find parameters for basic modeling functions
        • Linear
        • Sine
        • Power
        • Exponential
        • Logarithmic (note)
    3. Objectives
      1. Given a description of a real-world scenario, be able to identify the appropriate basic modeling function
      2. Given data, select parameters of the basic modeling functions that “fit” the data
    4. Reading: Chapter 7

  7. Slope function

    1. NTI:
      1. How do broaden the idea of describing the slope of a line to a general function?
      2. No \(h\) at this point
      3. The slope function is a function \(s()\) which is built from a function \(f()\)
    2. Topics
      1. Average rate of change. (Example: Quarterly return of stocks. HW of tree harvesting.) (Example 2: Sine wave with difference around 1 period. Or maybe this becomes low-pass filter example in accumulation.)
      2. \(s(x) \equiv \frac{f(x + 0.1) - f(x)}{0.1}\)
      3. Except for linear function, slope function depends on how big \(h\) is. So let’s fix \(h\) while we figure things out.
      4. Every function has a slope function.
      5. It’s helpful to name functions so we know where they come from. We’re going to use \(f'()\) to stand for the slope function of \(f()\)
    3. Reading: Chapter 9

  8. Composition Generally

    1. NTI:
      1. What temperature is it while driving up a mountain?
      2. Problem where they are calculating the slope of a sigmoid
    2. Topics:
      1. Formula for Hump (exponential with a quadratic)
      2. Driving up a mountain temperature vs altitude, altitude vs road position, road position vs time
      3. Hump is the slope function of Sigmoid
    3. Reading: Chapter 11

  9. Multiplication of functions

    1. NTIs:
      1. Multiplication about Sines, exponential, hump, sigmoid
    2. Topics:
      1. Multiplying functions of the same inputs gives a function of those inputs.
      2. Multiplying functions of different inputs gives a function whose inputs are the union of the inputs to the multiplicands.
    3. Be able to reverse engineer these products: sinexp, sinhump, sinsigmoid
    4. Reading: Chapter 11
