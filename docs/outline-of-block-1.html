Block 1 Functions and Mathematical Modeling

  1. Intro to the course

    1. NTI
      1. We can make the point about Linear Function is the model for change by drawing pictures or discussing how certain quantities change between two points in time
      2. The discussion about Calc Z should not be about “How is Z different”, but “Z is the new Calculus to which USAFA is transitioning; you may see differences between this course and the traditional course that some of your peers are taking.”
    2. Lecture Topics
      1. Calculus Z overview and trajectory
      2. Course resources and systems
      3. Calculus is the study of change
      4. Linear Function is the model for “change”
    3. Outcomes
      1. Understand than an equation relates quantities, but a function maps inputs to outputs
      2. Understand the difference between \(\equiv\) and \(=\)
      3. Be able to identify a function’s input
    4. Reading Chapter 1
    5. Assignments
      1. HW01
      2. Feedback01
      3. DD01

  2. Basics of modeling functions

    1. NTI
      1. Highlight the difference between \(e^x\) and \(e^kx\) and explain why it’s important I. Explain that the \(C\) in \(Ce^{kt}\) scales the output and the \(k\) scales the input
    2. Lecture Topics
      1. Naked modeling functions
      2. Basic Modeling functions and their behaviors based on their parameters
    3. Outcomes
      1. Memorize the names and mathematical representations of the six basic modeling functions
      2. Be able to identify the name of a function given in either math notation or R notation
    4. Readings: Section 1.3 and Chapter 3
    5. Assignments

  3. Functions as I/O and Notational Structure of functions (R<>Math)

    1. NTI
      1. Break student habit of using “x-axis” or “y-axis”; describe these as “horizontal” or “vertical” axes
    2. Topics
      1. Graphics and graphs
      2. Introduce Multi-input functions notation
      3. Distinguish between mathematical symbols: \(=\), \(\equiv\), $, <- See Secton 4.8
    3. Outcomes
      1. Be able to translate between traditional math notation and R expressions
      2. Be able to read contour plots and function tables showing inputs and outputs
    4. Reading: Chapter 4 & 5
    5. To do:

  4. Parameterized functions as models (eqn, params, data, applications)

    1. NTI
      1. Introduce idea that input has meaning, and output has meaning relative to the input
      2. Give students a function. Then change the units of input to a different unit (e.g. feet go to Meters)
    2. Objectives:
      1. Linear
      2. Sine
      3. Power
      4. Exponential
      5. Logarithmic
    3. Reading: Chapter 6

  7. Slope function

    1. NTI:
      1. How do broaden the idea of describing the slope of a line to a general function?
      2. No \(h\) at this point
      3. The slope function is a function \(s()\) which is built from a function \(f()\)
    2. Topics
      1. Average rate of change. (Example: Quarterly return of stocks. HW of tree harvesting.) (Example 2: Sine wave with difference around 1 period. Or maybe this becomes low-pass filter example in accumulation.)
      2. \(s(x) \equiv \frac{f(x + 0.1) - f(x)}{0.1}\)
      3. Except for linear function, slope function depends on how big \(h\) is. So let’s fix \(h\) while we figure things out.
      4. Every function has a slope function.
      5. It’s helpful to name functions so we know where they come from. We’re going to use \(f'()\) to stand for the slope function of \(f()\)
    3. Reading: Chapter 8

  8. Composition Generally

    1. NTI:
      1. What temperature is it while driving up a mountain?
      2. Problem where they are calculating the slope of a sigmoid
    2. Topics:
      1. Formula for Hump (exponential with a quadratic)
      2. Driving up a mountain temperature vs altitude, altitude vs road position, road position vs time
      3. Hump is the slope function of Sigmoid
    3. Reading: Chapter 9

  9. Multiplication of functions

    1. NTIs:
      1. Multiplication about Sines, exponential, hump, sigmoid
    2. Topics:
      1. Multiplying functions of the same inputs gives a function of those inputs.
      2. Multiplying functions of different inputs gives a function whose inputs are the union of the inputs to the multiplicands.
    3. Be able to reverse engineer these products: sinexp, sinhump, sinsigmoid
    4. Reading: Chapter 9

  10. Multivariate functions

    1. NTIs:
      1. Ask students how they compute the slope of a multi-input function as best they can imagine how (homework?) Build their imagined function (for the slope) and plot the function
    2. Outcomes:
      1. Be able to read & construct (in R) Contour Plots
      2. Be able to read & construct (in R) Surface Plots
      3. Be able to read a table w/ 2 inputs
    3. Reading: Chapter 10

  11. Piecewise functions [jettison if needed in favor of doing this as homework]

      1. Construct and evaluate piecewise functions
      2. Demonstrate how to implement in R
    2. Outcomes:
      1. Be able to identify break(s) of a piecewise function
      2. Be able to evaluate a piecewise function at a given input
      3. Be able to implement a piecewise function in R
    3. Reading: Chapter 11
